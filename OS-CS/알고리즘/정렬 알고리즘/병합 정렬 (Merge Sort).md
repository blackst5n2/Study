분할 정복(Divide and Conquer) 방식을 사용하여 항상 안정적으로 $O(N \log N)$의 시간 복잡도를 보장하는 정렬 알고리즘임.
# 병합 정렬 (Merge Sort)의 원리
병합 정렬은 크기 두 단계의 재귀적 과정으로 이루어짐.

#### 1단계: 분할 (Divide)
- 정렬해야 할 배열을 더 이상 나눌 수 없을 때(원소 1개)까지 계속 **절반으로 나눔**.
- 원소 1개짜리 배열은 그 자체가 정렬된 상태로 간주됨.

#### 2단계: 병합 (Merge)
- 분할된 인접한 두 부분 배열을 비교하여 **하나의 정렬된 배열** 로 합침.
- 이 합치는 과정이 핵심이며, **정렬된 상태를 유지하면서 합치는 것** 이 중요함.

# 병합 정렬의 특징과 효율성

| 구분              | 특징                                                                    |
| --------------- | --------------------------------------------------------------------- |
| 시간 복잡도          | 최선, 평균, 최악 모두 $O(N \log N)$                                           |
| 공간 복잡도          | $O(N)$ (임시 배열을 위한 추가 공간 필요)                                           |
| 안정성 (Stability) | 안정 정렬 (Stable Sort: O)                                                |
| 장점              | 데이터 상태와 관계없이 항상 **$O(N \log N)$** 을 보장함. **안정 정렬** 이므로 복합 조건 정렬에 유리함. |
| 단점              | 정렬 과정에서 임시 배열 $O(N)$ 크기가 필요하여 **추가 메모리(공간)** 를 사용함.                   |

#### 시간 복잡도($O(N \log N)$) 분석
- 분할 과정: 배열을 반으로 나누는 작업은 $O(\log N)$의 단계로 이루어짐.
- 병합 과정: 각 단계에서 $N$개의 원소를 병합하는 데 $(O(N))$의 시간이 소요됨.
- 총 시간 복잡도는 이 두 과정의 곱인 $O(N \log N)$이 됨. 이는 비교 기반 정렬이 달성할 수 있는 가장 효율적인 시간 복잡도
