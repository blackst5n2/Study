# 선택 정렬 (Selection Sort)의 원리
1. 전체 탐색: 배열의 **가장 앞쪽 위치(현재 정렬이 필요한 위치)** 를 기준으로 잡음.
2. 최솟값 찾기: 기준 위치부터 배열의 끝까지 탐색하여 **가장 작은 원소**를 찾음.
3. 교환: 찾은 가장 작은 원소를 기준 위치의 원소와 교환.
4. 반복: 기준 위치를 다음 칸으로 옮기고, 배열 전체가 정렬될 때까지 위 과정을 반복.

# 선택 정렬의 특징과 효율성

| 구분              | 특징                                          |
| --------------- | ------------------------------------------- |
| 시간 복잡도          | 최선, 평균, 최악 모두 $O(N^2)$                      |
| 공간 복잡도          | $O(1)$ (추가 메모리 거의 없음)                       |
| 안정성 (Stability) | 불안정 정렬 (Stable Sort: X)                     |
| 장점              | 구현이 매우 간단하고 직관적임. 데이터 교환 횟수가 $O(N)으로 매우 적음. |
| 단점              | 데이터가 이미 정렬되어 있어도 모든 원소르 ㄹ비교해야 하므로 비효율적임.    |

#### 시간 복잡도$(O(N^2))$ 분석
- 바깥쪽 루프(`i`): $N - 1$번 반복됨. (전체 크기$N$에 비례)
- 안쪽 루프(`j`): 매번 $N - i - 1$번 반복됨.
	- i=0일 떄 $N - 1$번
	- i=1일 때 $N - 2$번
	- ...
	- i=$N - 2$일때 1번
- 총 비교 회숫: $(N -1 ) + (N - 2) + \cdots + 1 = \frac{N(N-1)}{2} \approx O(N^2)$

데이터 상태에 관계없이 항상 모든 원소를 비교하므로, 시간 복잡도는 항상 $O(N^2)$로 고정됨.
