C/C++로 메모리 할당자를 직접 구현하다 보면 가장 무서운 게 Dangling Pointer나 Double Free 같은 메모리 버그임.
- 현대적 해법: "컴파일 타임에 메모리 실수를 100% 잡아내겠다."
- 원리: Rust 언어는 **소유권(Ownership)** 과 **빌림(Borrowing)** 이라는 개념을 도입했음. 메모리의 주인은 단 하나여야 하며, 주인이 사라지면 메모리는 자동으로 해제됨.
- 연결: Allocator들의 규칙을 언어 차원에서 강제하는 것. Garbage Collector(GC) 없이 C++만큼 빠르면서도 안전하기 때문에, 최근 리눅스 커널 개발 언어로도 채택됨.
