디스크 I/O가 느리고 순차 접근이 중요하다고 앎. 하지만 최신 NVMe SSD는 이야기가 다름.
- 고전적 한계: 과거의 알고리즘(B+ Tree 등)은 하드디스크(HDD)의 느린 헤드 이동을 최소화하도록 설계되었음.
- 현대적 변화: NVMe는 엄청나게 빠르고 병렬 처리가 가능함. 이제는 디스크의 물리적 한계보다 **소프웨어 오버헤드(Lock, System Call)** 가 더 큰 병목이 됨.
- 결과: 락을 최소화하는 Lock-free 자료구조나, 커널을 거치지 않고 유저 모드에서 직접 SSD에 접근하는 SDPK(Storage Performance Development Kit) 같은 기술이 각광받고 있음.
