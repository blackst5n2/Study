쉘코드 그 자체는 `open`, `mkdir`, `move` 같은 기본 명령어(유틸리티)가 아님. 쉘코드는 이러한 명령어를 실행시킬 수 있는 **가장 기초적인 프로그래밍 명령어들의 집합 (기계어)**.

# 쉘코드의 역할과 목표
쉘코드라는 이름은 해킹에 성공했을 때 **"쉘(Shell, 터미널환경)"** 을 띄어서 해커가 원하는 명령을 내리는 것이 **가장 흔한 목표** 였끼 때문에 붙여진 이름.

| 구분    | 쉘코드(Shellcode)                                             | 명령어 (`ls`, `mkdir`)                  |
| ----- | ---------------------------------------------------------- | ------------------------------------ |
| 본질    | 기계어 (Assembly) 코드                                          | 외부 프로그램 (바이너리)                       |
| 목표    | CPU에게 **"System Call"** 을 요청하여 OS 기능을 실행                   | OS에게 파일 조작 등 고수준 작업을 요청              |
| 작동 원리 | 1. 레지스터에 특정 번호(System Call 번호)를 세팅함. 2. `syscall` 명령을 실행함. | 쉘이 `fork`와 `exec`를 이용해 별도 프로그램을 실행함. |
| 예시    | `execve` 시스템 콜을 호출하여 `/bin/sh`를 실행하는 코드                    | `mkdir /tmp/hacked`를 실행하느 ㄴ것         |

결론: 해킹의 목표는 쉘코드를 메모리에 심고 실행시켜서, 최종저긍로 **운영체제의 시스템 호출(System call)** 을 통해 터미널 환경(쉘)을 띄우는 것. 쉘을 얻으면 해커는 그제야 `mkdir`, `rm` 같은 명령어를 입력할 수 있음.
