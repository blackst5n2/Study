**그래프(Graph)** 는 현실 세계의 복잡한 연결(네트워크, 지도, 소셜 관계 등)을 가장 잘 표현하는 자료구조이며, 그 위에서 동작하는 탐색 알고리즘은 **최적의 해법** 을 찾는 핵심 엔진.

# 그래프의 구조: 노드와 간선
그래프는 **정점(Vertex)** 또는 **노드(Node)** 와 이들을 연결하는 **간선(Edge)** 으로 구서어된 자료구조임.

#### 그래프의 종류
| 종류                         | 특징                             | 예시                     |
| -------------------------- | ------------------------------ | ---------------------- |
| 방향 그래프 (Directed Graph)    | 간선에 방향이 있어 A -> B로만 이동 가능.     | 네트워크 패킷 흐름, 웹사이트 링크 구조 |
| 무방향 그래프 (Undirected Graph) | 간선에 방향이 없어 A <-> B로 양방향 이동 가능. | 소셜 네트워크 친구 관계, 도로망.    |
| 가중치 그래프 (Weighted Graph)   | 간선에 비용(Cost)이 부여됨.             | 도로망의 거리, 네트워크의 지연 시간.  |

#### 그래프의 표현 (메모리 구조)
그래프는 실제 코드에서 주로 두 가지 방식으로 표현됨.
1. 인접 행렬 (Adjacency Matrix): $N \times N$ 행렬을 사용하여 노드 간 연결 여부를 저장함. (연결되어 있으면 1, 아니면 0).
	- 장점: 두 노드 간 연결 여부를 $O(1)$에 알 수 있음.
	- 단점: 노드가 많으면 메모리 낭비가 심함. (밀집 그래프 적합)
2. 인접 리스트 (Adjacency List): 각 노드에 연결된 노드 목록(리스트)을 저장함.
	- 장점: 노드 수가 적고 간선이 드문 그래프(희소 그래프)에서 메모리 효율이 좋음. (현실 그래프의 표준)

# 기본 탐색: BFS와 DFS
데이터를 검색할 때 어떤 순서로 노드를 방문할지 결정하는 가장 기본이 되는 알고리즘.

#### BFS (Breadth-First Search, 너비 우선 탐색)
- 원리: 시작 노드에서 가까운 노드들을 먼저 탐색함. (수평적 탐색)
- 구현 자료구조: **큐(Queue)** 를 사용하여 다음에 방문할 노드의 순서를 FIFO로 관리함.
- 활용: 최단 경로를 찾을 때 사용됨. (가중치가 없는 그래프에서 최단 경로는 곧 노드의 개수가 가장적은 경로이기 때문임.)
	- 예: 미로 찾기, 네트워크에서 최소 홉(Hop) 수 경로 찾기.

#### DFS (Depth-First Search, 깊이 우선 탐색)
- 원리: 하나의 경로를 따라 **갈 수 있는 가장 깊은 곳까지** 탐색한 후, 막히면 되돌아와(Backtrack) 다른 경로를 탐색함. (수직적 탐색)
- 구현 자료주고: 스택(Stack) 또는 재귀 호출을 사용하여 탐색 순서를 LIFO로 관리함.
- 활용: 모든 노드를 방문해야 할 때,  특정 경로가 존재하는지 확인할 때, 위상 정렬(Topological Sort) 등에 사용됨.
	- 예: 파일 시스템에서 특정 파일을 찾기 위한 검색, 웹 크롤러.

# 최단 경로 알고리즘 (Shortest Path)
가중치 그래프에서 시작 노드부터 목표 노드까지의 **가장 저렴한 비용(최단 거리, 최소 지연 시간)** 을 갖는 경로를 찾는 알고리즘.

#### Dijkstra 알고리즘 (다익스트라 알고리즘)
- 목표: 하나의 시작 노드에서부터 다른 모든 노드까지의 최단 거리를 찾음.
- 원리:
	1. 시작 노드의 거리를 0으로 설정하고, 나머지는 무한대로 설정함. 
	2. **우선순위 큐(최소 힙)** 를 사용하여 **가장 거리가 짧은(가중치가 낮은) 노드** 를 반복적으로 선택함.
	3. 선택된 노드를 경유하여 이웃 노드까지의 거리를 갱신함. (이미 저장된 거리보다 짧으면 갱신)
- 시스템 연결고리: 네트워크 라우팅 프로토콜(OSPF 등)의 핵심 원리이며, GPS에서 최단 경로를 찾는 데 사용됨.

#### A* 알고리즘 (A-star)
- 목표: 특정 시작 노드에서 특정 목표 노드까지의 최단 경로를 찾음. (Dijkstra는 '모든' 노드에 대한 최단 거리를 찾는 반면, A\*는 '특정' 목표에 초점)
- 원리:
	- Dijkstra의 '실제 거리'에 **휴리스틱(Heuristic)** 이라는 **'목표까지의 추정 거리'** 를 더하여 탐색 우선순위를 결정함.
	- 휴리스틱: "저 경로가 목표에 더 가까울 것 같다"는 직관적인 추정치. (예: 지도에서 목표 지점까지의 직선 거리)
- 성능: 휴리스틱 덕분에 불필요한 노드 탐색을 줄여 **Dijkstra보다 훨씬 빠르게** 퇴적의 경로를 찾음. (게임 AI, 복잡한 지도 탐색에 사용)
