타이머(Timer)/PWM은 제어의 **출력(Output)** 이었다면, **PID 제어** 는 그 출력을 결정하는 **두뇌(Algorithm)**.

# 제어 시스템의 꽃: PID 제어기 (Controller)
임베디드 시스템에서 모터 속도 제어, 드론 자세 제어, 온도 조절 등, 목표 값과 현재 값 사이의 오차를줄여나가는 모든 과정에 PID 제어기가 사용됨.

PID는 목표(Set Point)와 현재 상태(Process Value)의 **오차(Error)** 를 분석하여, 얼마나 강하게 제어해야 할지 결정하는 알고리즘.

#### P항 (Proportional Term, 비례)
- 원리: 현재의 오차 크기에 비례하여 제어력을 가함. 오차가 클수록 강하게, 작을수록 약하게 제어함.
- 문제점: 오차를 완전히 0으로 만들지 못하고 **잔여 오차(Steady-State Error)** 가 남음. (마치 목표 지점 앞에서 멈칫하는 것과 같음.)

#### I항 (Integral Term, 적분)
- 원리: **시간이 지남에 따라 누적된 모든 오차(과거)** 를 합산하여 제어에 반영함.
- 역할: P항만으로는 남는 **잔여 오차(Steady-State Error)** 를 시간이 지나면서 강제로 0으로 만들어줌.
- 문제점: 너무 크게 설정하면 과거의 잘못을 너무 오래 기어갛여 시스템이 심하게 흔들리는 (Overshoot/Undershoot) 현상을 유발함.

#### D항 (Derivative Term, 미분)
- 원리: **오차가 변하는 속도(미래)** 에 비례하여 제어력을 가함. 오차가 급격히 커지면 미리 제동을 검
- 역할: 시스템의 반응 속도를 향상시키고, 오버슈트(Over-shoot)를 줄여 안정성을 높임.
- 문제점: 미분은 노이즈(Noise)에 민감하여, 센서의 미세한 노이즈도 큰 오차로 증폭시켜 시스템을 불안하게 만들 수 있음.

#### 임베디드 연결 (PWM 출력)
PID 제어기가 계산한 최종 출력 값은 결국 모터의 속도나 밸브의 개도를 결정함. 이 값은 Timer/PWM의 Duty Cycle 레지스터 값으로 바로 전달되어 하드웨어를 제어함.

# 수학의 기본 언어: 선형 대수 (Linear Algebra)
선형 대수는 단순한 수학이 아니라, 데이터, 3D 그래픽, 머신러닝(ML) 등 현대 컴퓨터 공학의 모든 영역에서 데이터를 표현하고 조작하는 언어.

#### 벡터와 행렬: 데이터를 다루는 그릇
- 벡터 (Vector): 여러개의 숫자를 한 줄로 나열한 것. (예: [온도, 습도, 압력] -> ML의 특성(Feature))
- 행렬 (Matrix): 벡터들이 모여 테이블을 이룬 것. (예: 사진, 데이터베이스, 신경망의 가중치(Weight))

#### 행렬 연산의 중요성
- 변환 (Transformation): 행렬 곱셉은 3D 객체를 회전시키거나, 데이터를 다른 공간으로 이동시키는 등의 변환을 의미함. (로봇의 팔다리 위치 계산, 그래픽 엔진의 기본)
- 머신러닝 연결: 신경망의 모든 계산(`입력*가중치`)은 행렬 곱셈임. GPU나 NPU(신경망 처리 장치)가 빠른 이유는 이 행렬 곱셈을 병렬로 처리하는 것에 특화되어 있기 때문.

	임베디드 연결: 임베디드에서도 가벼운 이미지 처리나 행렬 계산이 필요할 때, DSP(Digital Signal Processor) 기능을 활용하거나, ARM CMSIS 같은 라이브러리의 최적화된 선형 대수 함수를 사용함.
