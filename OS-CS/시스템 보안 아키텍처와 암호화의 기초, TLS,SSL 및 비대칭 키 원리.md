**암호화(Cryptography)** 와 **보안 아키텍쳐**.
# 보안의 3대 요소: CIA Triad
보안 시스템을 설계할 때 추구하는 가장 기본적인 목표는 세 가지.

| 요소  | 영어 명칭           | 목표                                 |
| --- | --------------- | ---------------------------------- |
| 기밀성 | Confidentiality | 허가된 사람만 데이터를 볼 수 있도록 암호화           |
| 무결성 | Integrity       | 데이터가 전송 중 위변조되지 않았음을 보장 (해시)       |
| 가용성 | Availability    | 시스템이 필요할 때 항상 접근 가능하도록 보장 (DoS 방어) |

# 암호화의 기초: 키 사용 방식에 따른 분류
데이터의 **기밀성(Confidentiality)** 을 확보하는 두 가지 주요 암호화 방식.

#### 대칭 키 암호화 (Symmetric-key Cryptography)
- 원리: 암호화할 때 사용하는 키와 복호화할 때 사용하는 키가 동일함.
- 특징:
	- 속도: 매우 빠름. (알고리즘이 단순)
	- 문제점: 통신을 시작하기 전에 안전하게 키를 교환하는 것이 어려움. (제3자가 키를 훔칠 위험)
- 예시: AES (Advanced Encryption Standard), DES.

#### 비대칭 키 암호화 (Asymmetric-key Cryptography)
- 원리: 키 쌍(Public Key + Private Key)을 사용함.
	- 공개 키 (Public Key): 외부에 공개해도 안전하며, 암호화에 사용됨.
	- 개인 키 (Private Key): 사용자 본인만 비밀리에 가지고 있으며, 복호화에 사용됨.
- 특징:
	- 속도: 매우 느림. (수학적 연산 복잡)
	- 장점: 공개 키만으로 안전하게 통신을 시작할 수 있음.
- 예시: RSA, ECC (Elliptic Curve Cryptography).

# 데이터 무결성: 해시 함수 (Hash Function)
해시 함수는 데이터의 **무결성(Integrity)** 을 보장하는 핵심 도구.
- 원리: 임의의 길이 데이터를 입력받아 **고정된 길이의 해시 값(Digest)** 을 출력.
- 특징:
	- 단방향성: 해시 값을 거꾸로 돌려 원본 데이터를 복원하는 것이 불가능해야 함.
	- 충돌 회피: 원본 데이터가 단 1비트라도 바뀌면 해시 값이 완전히 달라져야 함.
- 사용처: 비밀번호 저장, 전송된 파일의 위변조 여부 확인.
- 예시: SHA-256, MD5 (보안 취약으로 현재는 거의 사용 안 함).

# TLS/SSL: 두 암호화 방식의 결합 (Handshake)
**TLS (Transport Layer Security)** 는 웹 브라우저(클라이언트)와 서버 간의 통신을 암호화하는 표준 프로토콜. 느린 비대칭 키와 빠른 대칭 키의 장점만 취하여 사용.

#### TLS Handshake (키 교환 과정)
1. 시작: 클라이언트와 서버가 연결.
2. 인증: 서버는 클라이언트에게 공개 키가 포함된 디지털 인증서를 전송하여 '내가 이 서버가 맞다'고 신원을 증명. (비대칭 키 사용)
3. 키 교환: 클라이언트는 서버의 공개 키를 사용하여, 통신에 사용할 **실제 대칭 키(세션 키)** 를 암호화하여 서버에 보냄.
4. 세션 시작: 서버만 개인 키로 이 암호화된 세션 키를 복호화할 수 있음.이제 클라이언트와 서버는 동일한 세션 키를 공유하게 됨.
5. 통신: 이후 모든 데이터 통신은 이 **세션 키(대칭 키)** 를 사용하여 고속으로 암호화 됨.

#### TLS의 역할
TLS는 기밀성(암호화) 외에도 **인증(Authentication)** 과 **무결성(Integrity)** 까지 모두 보장함.
- 무결성 보장: 전송되는 모든 데이터에 MAC(Message Authentication Code) 해시 값을 첨부하여, 데이터가 중간에 위변조되었는지 확인함.

# 임베디드 보안 아키텍쳐 (하드웨어 보안)
임베디드 시스템(IoT 기기)에서 보안은 소프트웨어만으로는 부족하며, 하드웨어의 지원이 필수적.
- Secure Boot (보안 부팅): 부팅 시 MCU가 실행하는 **첫 번째 코드(Bootloader)** 가 제조사가 서명한 정품 코드인지 암호화된 해시 값을 이용해 확인함. (위변조된 펌웨어 실행 방지)
- TPM/HSM (보안 칩):
	- 역할: 암호화 키, 인증서, 장치 ID 등 민감한 정보를 안전하게 저장하고, 엄호화 연산을 전담하여 수행하는 전용 하드웨어 모듈임.
	- 장점: 키를 이 칩 안에 보관하면, 해커가 소프트웨어적으로 RAM이나 Flash를 덤프해도 키가 절대 유출되지 않음.

> 이 모든 지식을 적용할 첫 번째 임베디드 하드웨어(예: STM32, ESP32)를 선택하고, 위에 설계한 Task들과 TLS 통신, 그리고 간단한 ML 추론까지 직접 구현해보는 것.
