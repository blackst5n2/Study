
#### 처리 파트별 권장 설정 (M4)

| 파트      | 권장 코어 타입            | QoS 클래스                      | 비고                                       |
| ------- | ------------------- | ---------------------------- | ---------------------------------------- |
| 그래픽 처리  | GPU + P-Core        | `QOS_CLASS_USER_INTERACTIVE` | Metal API 사용 시 시스템이 알아서 관리함              |
| 오디오 처리  | P-Core (고정 권장)      | `QOS_CLASS_USER_INTERACTIVE` | 지연 시간(Latency)에 매우 민감, `Real-time` 정책 고려 |
| 네트워크 처리 | E-Core (효율 중심)      | `QOS_CLASS_UTILITY`          | 데이터는 대기 시간이 길어 고성능 코어 점유는 낭비             |
| 로직 처리   | P-Core / E-Core 유동적 | `QOS_CLASS_DEFAULT`          | 작업의 무거움에 따라 시스템이 분배                      |

#### 팁
실제로 오디오나 그래픽 처리를 위해 아주 정밀한 제어가 필요해지면, Affinity 대신 아래 방식을 사용하게 됨.
- 오디오: `thread_policy_set`에서 `THREAD_TIME_CONSTRAINT_POLICY`를 사용하여 실시(Real-time) 스레드로 승격시킴.
- 대량 연산: Affinity를 직접 계산하기보다 GCD(Grand Central Dispatch)의 `concurrent queue`를 활용해 시스템이 M4의 10개 내의 코어를 다 쓰게 만듦.
