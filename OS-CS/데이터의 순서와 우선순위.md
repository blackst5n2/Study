운영체제(OS)의 **스케줄링** 부터 네트워크의 **버퍼 관리** 까지, 시스템의 **흐름(Flow)** 을 제어하는 핵심 메커니즘.

# 스택 (Stack): 후입선출의 원리 (LIFO)
**스택(Stack)** 을 데이터를 쌓아 올리는 구조로, **가장 나중에 들어옴 것이 가장 먼저 나가는(Last-In, First-Out, LIFO)** 원칙을 따름.

#### 핵심 동작
- Push: 데이터를 스택의 **맨 위(Top)** 에 추가함.
- Pop: 스택의 **맨 위** 데이터를 제거하고 반환함.

#### 시스템 연결고리: 함수 호출 스택
스택은 컴퓨터 시스템애서 **함수 호출** 을 관리하는 데 필수적으로 사용됨.
- 함수 호출 (Call): 함수가 호출될 때마다 **복귀 주소(Return Address), 매개변수, 지역 변수** 등의 정보가 **콜 스택(Call Stack)** 에 **(Push)** 됨.
- 함수 반환 (Return): 함수가 종료되면 해당 정보가 스택에서 **Pop** 되어 제거되고, 저장된 복귀 주소로 돌아가 이전 함수의 실행을 이어감.
- Stack Overflow: 함수가 끝없이 재귀적으로 호출되어 스택 메모리 영역을 초과할 때 발생함. (하드웨어 학습에서 다룬 스택 영역 침범과 연결됨,)

# 큐 (Queue): 선입선출의 원리 (FIFO)
**큐(Queue)** 는 데이터를 일렬로 세우는 구조로, **가장 먼저 들어온 것이 가장 먼저 나가는(First-In, First-Out, FIFO)** 원칙을 따름.

#### 핵심 동작
- Enqueue: 데이터를 큐 **맨 뒤(Rear)** 에 추가함.
- Dequeue: 큐의 **맨 앞(Front)** 데이터를 제거하고 반환함.

#### 시스템 연결고리: 스케줄링 및 버퍼 관리
큐는 **순서대로 공정하게** 처리해야 하는 시스템에 광범위하게 사용됨.
- OS 스케줄링: 운영체제의 **준비 큐(Ready Queue)** 는 CPU를 기다리는 프로세스들을 FIFO 또는 Round Robin 방식으로 관리함.
- 네트워크 버퍼: 네트워크 패킷이 도착하거나 전송되기 위해 임시로 대기하는 **버퍼(Buffer)** 는 보통 FIFO 큐로 구현되어 순서대로 처리됨.
- 그래프 탐색 (BFS): **너비 우선 탐색(Breadth-First Search)** 알고리즘에서 다음에 방문할 노드를 기록하는 데 사용됨.

# 우선순위 큐와 힙 (Heap): $O(\log N)$의 비밀
**우선순위 큐(Priority Queue)** 는 FIFO나 LIFO와 달리, 데이터가 들어온 순서와 상관없이 **우선순위가 가장 높은 데이터** 가 가장 먼저 나가는 추상적인 자료구조임. 이 우선순위 큐를 효율적으로 구현하는 자료구조가 바로 **힙(Heap)** 임.

#### 힙의 구조와 속성
힙은 **완전 이진 트리(Complete Binary Tree)** 의 형태를 가지며, 항상 다음 두 가지 **힙 속성(Heap Property)** 중 하나를 만족해야 함.
1. 최대 힙 (Max Heap): 부모 노드의 키 값이 **자식 노드의 키 값보다 항상 크거나 같음.** (가장 큰 값이 루트에 있음)
2. 최소 힙 (Min Heap): 부모 노드의 키 값이 **자식 노드의 키 값보다 항상 작거나 같음.** (가장 작은 값이 루트에 있음)

#### 힙의 성능 비밀: $O(\log N)$
힙은 데이터를 **배열** 에 저장하여 구현하므로, 배열 인덱스만으로 부모/자식 노드를 빠르게 찾을 수 있음.
- 삽입 및 삭제: 데이터를 삽입하거나 제거할 때, 힙 속성을 유지하기 위해 **루트부터 리프 노드까지 (또는 그 반대) 트리의 높이** 만큼만 비교 및 교환(Heapify)이 일어남.
- 결과: $N$개의 노드를 가진 완전 이진 트리의 높이는 $\log_2N$이므로, 힙의 모든 주요 연산은 **$O(\log N)$** 의 시간 복잡도를 가짐. (가장 큰/작은 원소를 $O(1)$에 찾지만, 제거/삽입은 $O(\log N)$ 소요)

#### 시스템 연결고리
힙은 **'가장 최적의 것을 반복적으로 찾아야 하는'** 문제에 최적임.
- Dijkstra/Prim 알고리즘: 최단 경로/최소 비용을 찾는 과정에서 **가장 작은 가중치를가진 노드** 를 반복적으로 선택하는 데 최소 힙이 사용됨.
- OS 태스크 우선선위: 실시간 운영체제(RTOS) 등에서 **가장 높은 우선순위의 태스크** 를 즉시 실행하기 위해 사용됨.
- 힙 정렬 (Heap Sort): $O(N \log N)$의 성능을 가지는 효율적인 정렬 알고리즘.
