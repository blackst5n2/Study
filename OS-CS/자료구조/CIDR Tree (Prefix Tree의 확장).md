- 진화 배경: 인터넷 라우터는 들어온 패킷의 IP 주소를 수십만 개의 라우팅 테이블에서 가장 긴 접두사가 일치하는 경로를 찾아야 함. (트라이 응용)
- 핵심 원리:
	- IP 주소를 32비트(IPv4) 또는 128비트(IPv6)의 이진 문자열로 보고, 이를 **이진 트라이(Binary Trie)** 로 저장함.
	- CIDR 표기법(192.168.1.0/24)의 접두사 길이에 따라 이 트라이를 압축하여 메모리 접근 횟수를 줄임.
	- 라우터는 이 트리를 따라 $O(32)$ 또는 $O(128)$의 매우 짧은 시간 안에 최적의 경로를 찾음.
- 결과: 네트워크 패킷을 초당 수백만 개씩 처리해야 하는 라우터에서 필수적인 고속 탐색 구조임.
