- 진화 배경: T-Tree처럼 캐시를 직접 고려하여 구현하면 다른 CPU 아키텍처(다른 캐시 크기)에서 비효율적이 될 수 있음.
- 핵심 원리:
	- 분할 정복(Divide and Conquer) 방법을 사용하여 알고리즘을 설계함.
	- 데이터를 재귀적으로 계속 분할하여, 결국 현재 CPU의 캐시 메모리에 들어갈 만큼 작은 크기가 되도록 만듦.
- 결과: 알고리즘이 특정 캐시 크기에 의존하지 않고, 어떤 CPU 환경에서도 자동으로 캐시를 최대한 활용하게 됨.
- 응용: 정렬(Sorting), 행렬 연산, 캐시 효율이 중요한 모든 기본 연산의 최적화.
