- 진화 배경: 일반 트라이(Trie)는 문자열이 길 때 노드가 너무 많아져 공간 효율성이 떨어짐. Merkle Tree는 데이터 블록 간의 관계만 확인.
- 핵심 원리:
	- 트라이(Trie) 구조를 기반으로 함.
	- 자식이 하나인 중간 노드들을 압축하여 하나로 합침. (예: "c-a-r" 노드를 하나의 노드로)
	- 이를 Merkle Tree처럼 각 노드에 해시 값을 부여하여 Patricia Merkle Tree를 만듦.
- 결과: 공간 효율성이 극대화되고, 데이터의 존재 여부와 무결성을 동시에 빠르게 검증할 수 있음.
- 응용: 이더리움(Ethereum)과 같은 블록체인의 **상태(State)** 를 저장하고 검증하는 데 사용되는 핵심 구조임.
