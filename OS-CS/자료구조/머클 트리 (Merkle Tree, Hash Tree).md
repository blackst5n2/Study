# 분산 및 무결성 검증 자료구조
대규모 분산 시스템(블록체인, 클라우드 스토리지, 빅데이터)에서는 데이터의 무결성(Integrity) 검증과 고속 검색이 핵심임.

#### 머클 트리 (Merkle Tree, Hash Tree)
- 목표: 방대한 양의 데이터 블록 중 일부 데이터가 변조되었는지를 빠르게 검증하는 것. (블록체인의 근간)
- 구조: 이진 트리 형태임.
	- 리프 노드: 실제 데이터 블록의 해시 값 (예: SHA-256)을 가짐.
	- 내부 노드: 두 자식 노드의 해시 값을 합쳐서 다시 해시한 값을 가짐.
	- 루트 노드: 최종 해시 값 (Merkle Root)을 가짐.
- 장점: 전체 데이터(GB 단위)를 모두 전송할 필요 없이, Merkle Root와 변조가 의심되는 경로상의 소수 해시 값만으로 무결성을 $O(\log n)$ 시간에 검증할 수 있음.
